# Cursor Agent Orchestrator — Patterns

## Full multi-turn cycle

```
1. cursor_agent_spawn(task: "...")
   → response includes session_id + watcher command
2. Bash(run_in_background: true): <watcher command from response>
3. END TURN. Do other work, answer user questions.
4. [watcher completes, question auto-surfaces]
5. cursor_agent_reply(message: "dig into X next")
   → response includes new watcher command
6. Bash(run_in_background: true): <new watcher command>
7. END TURN. Do other work.
8. [watcher completes again]
9. cursor_agent_reply(message: "Done. You can stop now.")
```

The watcher command is generated by the MCP with the correct bridge directory and turn number. Always use the command from the response — don't construct your own.

## Auto-Reply Loop (for working sessions)

For working sessions where the agent iterates through many steps and you just need to say "continue". Uses the orchestrator CLI directly:

```bash
Bash(run_in_background: true):
ORC=/path/to/orchestrate-cursor-agent-mcp/src/orchestrator.js
QNUM=1; MAX=22; START=$(date +%s)
while [ $QNUM -le $MAX ]; do
  [ $(( $(date +%s) - START )) -gt 300 ] && break
  STATUS=$(node $ORC check 2>/dev/null)
  echo "$STATUS" | grep -q '"pending": true' || { sleep 2; continue; }
  MSG=$(echo "$STATUS" | python3 -c "import sys,json; print(json.load(sys.stdin)['message'])" 2>/dev/null)
  echo "Q${QNUM}: ${MSG}"
  [ $QNUM -eq $MAX ] && node $ORC reply "All done. You can stop now." \
                      || node $ORC reply "continue to Q$((QNUM+1))"
  QNUM=$((QNUM+1))
done
echo "Answered $((QNUM-1)) questions"
```

Check progress: `TaskOutput(task_id, block: false)`.

## Crash Recovery

If the watcher completes with an error or `cursor_agent_status` shows `completed` too early:

1. Call `cursor_agent_result()` to see stderr/stdout and diagnose
2. Spawn a new agent with full context (new sessions have NO memory):

```
cursor_agent_spawn(task: """
CONTEXT FROM PREVIOUS SESSION:
- Completed: Q1 through Q8
- Results written to: /tmp/research-output.md (15KB)
- Remaining: Q9 through Q22

RESUME: Start from Q9. Append to /tmp/research-output.md (do not overwrite).
""")
```

## Research Pattern (write to file, short bridge messages)

For research tasks that produce large output, tell the agent to write to a file and send only short summaries through the bridge:

```
cursor_agent_spawn(task: """
Research [topic]. For each finding:
1. APPEND details to /tmp/research-output.md
2. Call report_to_orchestrator with only a 1-line summary
3. Wait for my reply before continuing

Start by asking what to research.
""")
```

Verify file growing: `stat -f "%z" /tmp/research-output.md`
